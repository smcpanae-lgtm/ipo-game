# 第1弾設計書：攻守トレードオフ＋IPOウィンドウ（叩き台 v1）

作成日: 2026-06-11 ／ ステータス: **監修待ち**

## 1. 設計のねらい

「無難な選択をすれば必ず上場できる」構造を壊す。

- 守りの選択（管理体制の整備）には **成長鈍化・コスト増** の代償を付ける
- 攻めの選択（成長優先）には **成長率アップ** の見返りと **リスクフラグ（爆弾）** を付ける
- 上場には「審査通過（守り）」と「形式要件＝時価総額・流通株式（攻め）」の **両方** が必要なため、
  どちらかに全振りすると必ず行き詰まる

## 2. 新メカニクス

### 2-1. 攻守バランスメーター（サイドバー常時表示）

- ⚔ 攻め度: 成長系選択の累積
- 🛡 守り度: 統制系選択の累積（既存の準備スコアと連動）
- どちらかが極端に偏ると警告表示（IPO顧問が苦言を呈する）

### 2-2. 成長率への介入（トレードオフの実装方法）

- `growth_tax`: 守りの選択 → 四半期成長率を一時的に -1〜-3pt（現場負荷を表現）
- `growth_boost`: 攻めの選択 → +2〜+4pt ＋ リスクフラグ／爆弾
- 時価総額は成長率の複利で決まるため、N期の形式要件判定に直結する

### 2-3. 形式要件の実効化

- N期審査時に時価総額・流通株式時価総額が市場基準未達なら、
  審査内容が良くても **「形式要件未達」で上場不可**（現状より厳格化）
- 基準: グロース 流通時価総額5億 ／ スタンダード10億 ／ プライム100億（既存表示と整合）

### 2-4. IPOウィンドウ（市況変動）

- 市況指数 0〜100。毎四半期 ±15 の範囲で変動（モメンタム付き：上げ/下げが続きやすい）
- サイドバーに「📈 市況: 強気/中立/弱気」メーターを表示
- 時価総額評価に乗数: `×(0.70 + 市況/100 × 0.60)` → 弱気32なら×0.89、強気80なら×1.18
- N期に「申請タイミング」イベントを追加: 今Q申請する／1Q見送る（最大2回）
  - 見送り: 準備改善のチャンス＋市況回復への賭け。ただし悪化もあり得る
  - 弱気（30未満）で申請: 公開価格ディスカウント・株主数集めが困難
  - 強気（70超）で申請: 評価額プレミアム

## 4. バランス目標（チューニング指針）

- 「全部守り」プレイ → N期に時価総額が要件の **約8割** で頭打ち → 形式要件未達で延期
- 「全部攻め」プレイ → 爆弾2〜3個が確定発動 → 審査不通過
- 最適解は「N-3〜N-2で攻め、N-1で守りへシフト」＝実務のセオリーと一致させる（学習効果）
- 市況も含めた成功率目標: 初見プレイヤーで 30〜40%、2周目で 60〜70%

## 5. 実装結果（2026-06-11 実装完了・オートプレイ検証済み）

- 成長率修正（恒久/一時）・市況指数・時価総額乗数・攻守メーター・市況メーター・
  IPOウィンドウ判断（弱気市況で申請強行/3ヶ月延期）・グロース審査「高い成長可能性」チェックを実装
- 主な恒久成長税：労務是正-2pt / J-SOX本格-2pt / 管理優先(組織)-2pt / 発生主義-1pt（CFO採用は+1pt）
- オートプレイ結果（SaaS×グロース、3シード）:
  - 全A（無難・勤勉）: 1/3 成功 — 残り2回は リスク蓄積・AGM否決起因の指摘で延期
  - 手抜き（常に最終選択肢）: 0/3 成功（不通過/解任/倒産）
  - ※最終的な難易度調整は人間のプレイテストで行う（growth_tax の pt 数・成長性チェックの 65% が主な調整ノブ）
